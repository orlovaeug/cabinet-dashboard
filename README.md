## Quick start (local)

```bash
# 1. Clone
git clone https://github.com/YOUR_USERNAME/nl-cabinet-dashboard
cd nl-cabinet-dashboard

# 2. Install Python deps (only `requests`)
pip install -r requirements.txt

# 3. Fetch data  (takes ~30 seconds, polite crawl rate)
python backend/fetch_data.py

# 4. Open dashboard вЂ” serve locally so fetch() works
cd frontend
python -m http.server 8080
# в†’ open http://localhost:8080
```

> **Why a local server?** Browsers block `fetch('data/dashboard.json')` when
> opening `index.html` directly as a `file://` URL. The `python -m http.server`
> one-liner fixes this with zero setup.

## Free hosting on GitHub Pages

1. Push the repo to GitHub.
2. Go to **Settings в†’ Pages в†’ Source** and set:
   - Branch: `main`
   - Folder: `/frontend`
3. GitHub Pages will serve `frontend/index.html` at  
   `https://YOUR_USERNAME.github.io/nl-cabinet-dashboard/`

The **GitHub Actions** workflow (`.github/workflows/refresh.yml`) runs every day
at 06:00 UTC, calls `fetch_data.py`, and commits the new `dashboard.json`
automatically вЂ” **no server, no API key, no cost**.

### Enable Actions write permission
Go to **Settings в†’ Actions в†’ General в†’ Workflow permissions** and select
**"Read and write permissions"** so the bot can commit the JSON file.

## Features

| Feature | Description |
|---|---|
| **Minister Activiteit** | Card per ministry В· press release count (7d / 30d) В· activity status dot В· days since last publication |
| **Coalitie Breakdown** | PVV В· VVD В· NSC В· BBB press release totals + member list |
| **Trending Onderwerpen** | Word cloud from titles + introductions across all ministries; click a word to jump to filtered feed |
| **Persberichten** | Full searchable + filterable feed; filter by ministry, party or free text |

## Data sources

| Source | URL pattern | Notes |
|---|---|---|
| Official API | `opendata.rijksoverheid.nl/v1/infotypes/news/ministries/{slug}` | JSON, no auth |
| RSS feeds | `rijksoverheid.nl/ministeries/{slug}/nieuws.rss` | Supplement / fallback |

Both sources are merged and deduplicated by title.

## Updating the cabinet

Edit `MINISTRIES` in `backend/fetch_data.py` вЂ” each entry needs:
- `name` вЂ” display name
- `minister` вЂ” full name
- `role` вЂ” role title
- `party` вЂ” party abbreviation
- `party_color` вЂ” hex color for UI
- `rss` вЂ” RSS feed URL
